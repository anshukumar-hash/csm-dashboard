# Session Summary — 2026-05-26

Working notes for the dashboard rebuild + automation work landed across four
commits on `main` (`8360eb6` → `20a207a`). This doc captures the *why* so the
context survives the commit messages.

## What changed

### 1. Full gviz sync replaces Drive-MCP truncated extract
Drive MCP only returns ~85–95 rows per Vini-daily tab — the live sheet has
4,000+. While the source sheet was private, we lived with the truncated
snapshot. Once the sheet was set to **Anyone with link → Viewer**, we switched
to direct gviz fetch for all three tabs:

| Tab | gid | Rows pulled |
| --- | --- | --- |
| Vini daily metrics | `1616842841` | 4,043 |
| Payment master (CS_Vini_Stage) | `674556270` | 124 |
| CSAT readings | `701797891` | 23 |

Effect on the dashboard: 25 May appointment count jumped from 18 (Drive MCP
slice) to 111 (real total), matching what's visible in the sheet itself.

### 2. CSAT `headers=1` fix
Default gviz behavior auto-detected headers wrong on the CSAT tab — it returned
16 rows instead of the 22 actually present. Adding `&headers=1` to both the
build script (`fullsync_gviz.ps1` / `sync.ps1`) and the browser-side live-fetch
URL fixed the row count.

Note: of the 22 CSAT enterprises currently in the sheet, only **2** match
Vini Payment scope (Warren CDJR, CallSource). The remaining 43 Vini
enterprises show `—` in Communication RAG because no CSAT row exists for
them — that's a data-coverage gap, not a code bug.

### 3. GitHub Actions auto-sync every 15 min
Previously the dashboard relied on the browser-side `gviz` fetch which only
runs when the page is open. Now a cron-triggered workflow keeps the embedded
snapshot fresh even when nobody has the dashboard open:

- `.github/workflows/sync-dashboard.yml` — `*/15 * * * *` cron + manual dispatch
- `.github/scripts/sync.ps1` — portable PowerShell (uses `ConvertFrom-Json`,
  not the Windows-only `System.Web.Extensions`)
- Permissions: `contents: write` so the runner can push commits back
- Concurrency guard prevents overlapping runs from racing on `git push`
- `[skip ci]` in auto-commit messages prevents the workflow from triggering
  itself

No secrets needed — the sheet is public-link-viewable, gviz is a public
endpoint, and `GITHUB_TOKEN` is auto-injected.

### 4. Dashboard data fixes that came along with the sync

- Segment backfill chain: existing vini_stage → fresh v_rows by (rid, agent) →
  by enterprise name → Customer Type inference → manual EID override → `Other`
- Manual segment override for 4 enterprises that have no Vini-daily rows
  anywhere in the export (Next Gear, PalmEasy, Dutch Miller, Toronto Honda)
- Dynamic segment enumeration in grouped tables (was hardcoded list,
  silently dropped the `Other` row including its 1 churned agent)
- Agent-level view trimmed to Identity + Final Status + Usage + RoI only
  (hidden: Report Sent, Enterprise-Level Payment, Tickets, Communication,
  Leadership). New `hide-*` CSS classes for per-bucket toggling.
- Rooftop View: Churned column now shows a `Churned` badge when the agent
  stage is Churned. Communication RAG = blank `—` when no CSAT history.
